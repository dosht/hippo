### MVP-2-06: Fix Silent Gold-Drop Regression in extract.py is_duplicate()

**Severity:** High

**Reported:** 2026-05-01

**Regression window:** 2026-04-28 to present

---

## Bug Description

Since approximately 2026-04-28, the nightly Hippo extraction pipeline has produced zero new gold
entries despite processing hundreds of sessions per night. The gold corpus is frozen at 29 entries
(last mtime 2026-04-27). The manifest has accumulated 111 sessions since 2026-04-30 with
`status: "gold"` and `gold_paths: []` (empty array), meaning the model extracted valid entries but
they were silently discarded before being written to disk.

The bug went undetected for approximately 4 days because no monitoring surfaces the condition
"gold-status sessions with empty gold_paths."

---

## Steps to Reproduce

1. Take any session processed on or after 2026-04-30 that has `status: "gold"` and `gold_paths: []`
   in `~/src/hippo/manifest.jsonl` (111 such entries exist as of 2026-05-01).
2. Re-run `is_duplicate()` from `scripts/extract.py:428-486` against the entry. Use the qmd query
   that `is_duplicate()` constructs: `entry_id + topics` only (no body, no summary).
3. Observe that the top hit scores >= 0.85 on a topically-related but content-distinct entry.

**Reproduction evidence:** Replaying `is_duplicate()`'s qmd query for the Apr 30 entry
`mem-clock-based-subscription-expiry` returned `mem-billing-access-packages-source-of-truth.md`
as the top hit with score 0.93. These entries are not duplicates. The threshold is 0.85, so the
entry was rejected and `gold_paths` was left empty.

---

## Expected Behavior

- A newly extracted entry with a unique `id` (no existing `.md` file under that name in the gold
  directory) is written to disk and its path is recorded in `gold_paths`.
- Topically similar but content-distinct entries are not rejected.
- `manifest.jsonl` entries with `status: "gold"` have non-empty `gold_paths` unless the entry is
  a genuine exact-id collision.

---

## Actual Behavior

- Every entry on topics already covered by the existing gold corpus (billing, Supabase, Kenznote)
  is rejected by `is_duplicate()` with a false-positive score >= 0.85.
- `gold_paths` is set to `[]` for all rejected entries.
- No error or warning is logged. The pipeline reports success.

---

## Root Cause Analysis

`is_duplicate()` in `scripts/extract.py:428-486` calls the qmd reranker with a query built from
only `entry_id + topics` (3-5 generic tags). This query is too thin to discriminate content. As the
gold corpus grew to cover the user's most-trodden domains (billing, Supabase, Kenznote), every new
entry on those topics now matches an existing one at score >= `DUPLICATE_THRESHOLD` (0.85) regardless
of actual content overlap.

Relevant constants: `DUPLICATE_THRESHOLD = 0.85` and `QMD_DUPLICATE_CHECK_N = 3` at
`scripts/extract.py:57,63`.

The fundamental design mismatch: semantic deduplication belongs in the reconcile cold path
(`scripts/reconcile.py`, which has full entry bodies and corpus-wide context, and already outputs
`reconcile-clusters.jsonl` and `reconcile-proposals.jsonl`). Using it in the extract hot path with
a thin topic-only query introduces a false-positive surface that grows with corpus size.

---

## Fix Verification

### AC1: Deterministic duplicate check at extract time
- [x] `is_duplicate()` (or its replacement) contains no qmd or reranker calls.
- [x] Duplicate detection at extract time is based solely on exact entry-id collision: if
  `(gold_dir / f"{entry['id']}.md").exists()` then skip or merge, otherwise write.

### AC2: Stuck sessions recover
- [x] Re-running extract on the 111 manifest entries with `status: "gold"` and `gold_paths: []`
  produces non-empty `gold_paths` for entries whose id does not collide with an existing file.
- [x] Gold corpus grows beyond 29 entries after the fix is deployed.
  - Confirmed by tester 2026-05-01: corpus grew from 29 to 45+ entries after running --recover-empty-gold on 117 stuck sessions. qmd search for "clock-based subscription expiry" returns mem-clock-based-subscription-expiry.md at 92% score.

### AC3: Regression test coverage
- [x] A test in `tests/test_extract.py` covers: (1) new entry with no id collision is written and
  `gold_paths` is non-empty, (2) entry whose id matches an existing file is skipped, (3) a
  topically similar but content-distinct entry is written (regression guard for this exact bug).

### AC4: Silent-failure monitoring
- [x] Nightly summary output (or a periodic check) surfaces the count of `gold-status` sessions with
  empty `gold_paths` over the last 24 hours, so this class of regression cannot go undetected for
  more than one nightly run.

## Architect Review (2026-05-01)

**Status: approved_with_hints**

The plan is correct and well-scoped. Root cause is accurately diagnosed, fix is minimal and deterministic, and the architectural boundary (semantic dedup belongs in reconcile, not extract) is sound.

**Hints for implementation:**

1. `is_duplicate()` signature change (`proposed_entry, gold_dir`) is a breaking change to the function's public surface. Verify no other callers outside `extract_session()` exist before renaming the signature. `grep -rn "is_duplicate"` in `scripts/` is the check.

2. Task 2.1 extends `is_eligible()` via a flag passed from `main()`. The cleanest pattern is a second predicate function (e.g. `is_recoverable(entry)`) rather than a boolean argument threading through, keeping `is_eligible()` unchanged. Either approach is acceptable; flag-threading is also fine given the small codebase.

3. Task 3.1-3.3: the monitoring log should emit at `WARNING` level via `log.warning(...)` (the stdlib logger already configured), not via a bare `print()`. Confirm the existing `log = logging.getLogger(...)` instance is in scope at the end of `main()`.

4. Coding standards require type hints on all public functions. The new `is_duplicate(proposed_entry: dict, gold_dir: Path) -> bool` signature must include the return type. `gold_dir` should be typed as `pathlib.Path`, not `str`.

5. Test 4.6 (`test_main_warns_on_empty_gold_paths`) is the hardest to wire up — `main()` reads `manifest_path` from args. Use `caplog` (pytest's log capture fixture) rather than mocking `log.warning` directly; it integrates cleanly with the stdlib logger.

6. The safe-rewrite pattern (`write_text` via tmp + `os.replace`) documented in `coding-standards.md` applies to `write_gold_entry` if it rewrites existing files. Confirm that function already uses atomic write or add it if not.
