### S3: Git Post-Merge Hook for QMD Reindex

**As a developer**, I want a git post-merge hook that automatically runs `qmd update && qmd embed`
after every `git pull` so that new gold entries are immediately searchable on any machine.

#### Acceptance criteria

- [ ] File .git/hooks/post-merge exists and is executable (`chmod +x`)
- [ ] Hook contains: `#!/bin/bash` header and `qmd update && qmd embed`
- [ ] Tested: after `git pull` that introduces a new gold entry, `qmd query` returns the new entry
- [ ] Developer-guide documents the hook setup for new machines

#### Architect Notes

Wave 3a (parallel with S4 and S8.5). No stub files; `.git/hooks/post-merge` is not tracked in git and must be set up manually on each machine. The hook runs `qmd update && qmd embed`, the same command `extract.py` runs at the end of each run (S9); both must use the same QMD collection name (`hippo`) and mask. Document the setup command in the developer-guide so new machines can replicate it. Design doc ref: `docs/product/epics/MVP-1/design.md` S3.
