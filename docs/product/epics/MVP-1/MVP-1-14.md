### S14: Session Metadata Enrichment

**As Mustafa**, I want every ingested session to record its authoritative git branch,
working directory, and session start timestamp so that gold entries can be filtered and
tagged by branch, workspace, and real time of occurrence (not ingest time).

#### Context

Each Claude Code session JSONL carries, on every record, the exact `cwd`, `gitBranch`,
and per-record `timestamp` fields. These are currently not read during ingest, so the
manifest and downstream silver/gold lose provenance that is trivially available:

- `cwd` disambiguates project directories like `-Users-mu-Business-claude-team-main`
  (could decode to `/…/claude-team/main` or `/…/claude-team-main`). `cwd` is exact.
- `gitBranch` is a natural tag users will want to filter gold by (e.g. "show me
  learnings from the `payments` branch").
- `session_started_at` (timestamp of the first non-permission-mode record) is far more
  useful than `ingested_at` for chronological queries; `ingested_at` only says when
  Hippo noticed the file, not when the work happened.

#### Acceptance criteria

- [x] `ingest.py` reads the first ~20 records of each JSONL and captures:
  - `cwd`: from any record that has it (first seen)
  - `git_branch`: from any record that has `gitBranch` (first seen)
  - `session_started_at`: ISO8601 timestamp from the first record that has `timestamp`
- [x] These fields are written to every new manifest entry. Existing entries are not
  backfilled (backfill is out of scope; old entries fall through with null values).
- [x] `manifest.py` schema comment block documents the three new fields (type, source,
  nullability).
- [x] `compact.py` `_silver_frontmatter()` propagates `cwd`, `git_branch`, and
  `session_started_at` into the silver YAML frontmatter. Silver files without these
  fields (from older manifest entries) still compact without error.
- [x] `extract.py` reads the new silver frontmatter fields and passes them to the
  extraction prompt (via metadata injection, same pattern as existing `project` / `agent`).
- [x] Gold frontmatter schema docs (`gold/sample-gold-entry-format.md` and arc42) are
  updated to include optional `git_branch` and `session_started_at` fields on entries
  when the extract model deems them relevant.
- [x] Unit tests:
  - `extract_session_metadata()` helper covers: missing fields, all fields present,
    malformed JSON lines, file that only has a permission-mode record.
  - `_silver_frontmatter()` test covers the three new fields appearing / missing.
- [x] Developer-guide README gets a short note: "The manifest now records cwd,
  git_branch, and session_started_at; older entries may have null values."

#### Architect Notes

Pure schema extension. No change to pipeline topology. No change to existing statuses.
Backward compatible: all three new fields are nullable; old silver/manifest entries do
not need rewrite.

**Do NOT** attempt to derive `git_branch` by parsing `project_hash`. `gitBranch` in the
JSONL is authoritative.

**Do NOT** add `claude_version` — low information value per S14 discussion. Can be
added in a follow-up if needed.
