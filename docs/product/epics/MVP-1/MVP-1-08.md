### S8: Build compact.py

**As a developer**, I want scripts/compact.py to read bronze sessions and produce silver summaries
via `claude -p`, preserving near-miss failures and concrete configuration details, so that extract.py
has high-quality input.

#### Acceptance criteria

- [ ] `python scripts/compact.py` reads manifest entries with status "bronze", skips short sessions and memory_query sessions
- [ ] For each eligible session, calls `claude -p --model claude-sonnet-4-5 --max-turns 1 --no-mcp` with the compaction prompt from docs/playbooks/compact.md
- [ ] Silver output written to silver/YYYY-MM-DD_<session-id>.md
- [ ] Manifest entry updated: status "silver", compacted_at timestamp, silver_size_bytes
- [ ] Compaction ratio logged: if ratio is less than 40% or more than 90%, a warning is printed
- [ ] Running compact.py on already-silver sessions is a no-op (idempotent)
- [ ] Large sessions (content exceeding 80K tokens estimated by character count / 4) print a warning and are skipped rather than failing silently; these are noted in manifest as status "skipped-large"

#### Architect Notes

Wave 2 (parallel with S2). Stubs to flesh out: `scripts/compact.py` and `docs/playbooks/compact.md`. The `--no-mcp` flag on all `claude -p` calls is required; without it MCP servers load and add latency and side effects during bulk runs. The compaction prompt in `docs/playbooks/compact.md` must explicitly instruct the model to preserve near-miss failures (MemRL finding). Boundaries: do NOT modify `scripts/ingest.py` or `scripts/manifest.py` signatures. Gotcha: use `manifest.py` helpers for all JSONL I/O. Design doc ref: `docs/product/epics/MVP-1/design.md` S8.
