### S11: Scoped Pipeline Validation Run (one active project)

**As Mustafa**, I want the pipeline run against the sessions of a single currently-active
project (kenznote-payments) so that I can validate real-world value quickly and iterate on
prompts without burning my weekly Claude Max quota on full history.

#### Context

Original intent was a full-history ingest of all ~590 sessions. Capacity analysis
(Apr 18) showed that would consume a meaningful fraction of the weekly Claude Max
quota while the user is actively developing on the same account. Value can be
demonstrated with a scoped run against one currently-active project. Full-history
ingestion is deferred to Post-MVP.

Target project: `/Users/mu/Business/Kenznote/kenz-note/payments`
(project_hash `-Users-mu-Business-Kenznote-kenz-note-payments`, ~7 top-level sessions
+ their subagent transcripts).

#### Acceptance criteria

- [ ] `python -m scripts.ingest --source ~/.claude/projects/-Users-mu-Business-Kenznote-kenz-note-payments`
  run; all kenznote-payments sessions + subagents present in manifest.jsonl
- [ ] `python -m scripts.compact` run to completion on the kenznote-payments subset;
  any `skipped-large` sessions logged
- [ ] `python -m scripts.extract` run to completion; final gold entry count reported
- [ ] `qmd update && qmd embed` run; `qmd status` reports total document count
- [ ] `git add gold/ manifest.jsonl && git commit -m "kenznote-payments validation ingest: <date> - <N> gold entries" && git push` completes
- [ ] At least 3 gold entries produced from real kenznote-payments sessions (not counting
  hand-crafted seeds)
- [ ] Memory subagent query from a non-Hippo project returns at least one grounded
  answer sourced from an auto-extracted kenznote-payments entry
- [ ] Silver frontmatter includes `git_branch` and `session_started_at` (S14) and
  subagent transcripts were ingested and propagated with `agent_task` (S15)

#### Architect Notes

This story intentionally narrows scope from "full history" to "one active project" to
unblock real validation while respecting capacity constraints. It also depends on S14
(metadata enrichment) and S15 (subagent ingestion) so the validation exercises the
final data shape.

Out of scope (deferred to Post-MVP):
- Full-history ingestion of all ~590 top-level sessions
- Reconciliation pass on the resulting gold entries
- Cross-project multi-workspace validation
