### S2: QMD Collection Setup and Indexing

**As a developer**, I want the QMD collection registered and indexed against gold/entries/ so that
`qmd query` returns results and the memory subagent can search them.

#### Acceptance criteria

- [x] `npm install -g @tobilu/qmd` documented in developer-guide as a prerequisite
- [x] `qmd collection add ~/src/hippo/gold/entries --name hippo --mask "**/*.md"` runs cleanly
- [x] `qmd embed` completes and `qmd status` reports the correct document count matching gold/entries/
- [x] `qmd query "claude code session cdp port" --collection hippo --json -n 3` returns at least one match
- [x] QMD index files (.qmd/ or similar) are confirmed gitignored

#### Architect Notes

Wave 2 (parallel with S8). No new files; pure configuration and docs work. Confirm `.qmd/` is in `.gitignore` (already present after infra phase). Boundaries: do NOT create or modify gold entries (S1), do NOT touch `scripts/`. The `--mask "**/*.md"` flag on `qmd collection add` is critical; without it the sample template is indexed and inflates result counts. Run `qmd embed` to completion here (this is the first run, which downloads the ~2GB GGUF model). Design doc ref: `docs/product/epics/MVP-1/design.md` S2.
