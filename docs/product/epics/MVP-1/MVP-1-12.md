### S12: Minimal Pipeline Status Command

**As Mustafa**, I want a quick way to check pipeline health (session counts by status, gold entry
count, last run timestamp) without the full P3 dashboard, so that I know if the pipeline ran and
how many entries exist.

#### Acceptance criteria

- [ ] docs/playbooks/status.md is simplified to remove staleness, suggestions, and promotion candidate sections (those are P3 scope); MVP-relevant sections retained: pipeline state, gold entry stats, QMD health
- [ ] The following commands work and produce correct output from the project root:
  - `cat /Users/mu/src/hippo/manifest.jsonl | jq -s 'group_by(.status) | map({status: .[0].status, count: length})'`
  - `find /Users/mu/src/hippo/gold/entries -maxdepth 1 -name "*.md" | wc -l`
  - `qmd status`
- [ ] A one-liner alias or short script (scripts/status.sh) is added that runs all three and prints a summary matching the format in status.md

#### Architect Notes
