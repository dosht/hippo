### S6: Read-Path Smoke Test from Another Project

**As Mustafa**, I want to open a Claude session in the Transgate frontend project, invoke
hippo-remember, and receive a grounded answer from a gold entry so that I know the end-to-end
read path works across projects.

#### Acceptance criteria

- [ ] Claude session opened in ~/src/transgate-frontend (or equivalent non-Hippo project)
- [ ] hippo-remember skill visible in session (confirms global install worked)
- [ ] At least 2 queries issued: one that matches a gold entry and one that does not
- [ ] Matching query returns Answer with specific detail (port number, command, file path, etc.)
- [ ] Non-matching query returns "No relevant entries found" without fabrication
- [ ] Latency is acceptable (under 30 seconds per query including subagent spawn)

#### Architect Notes

Wave 3b (sequential after S3 and S5; this is a human validation gate). No file stubs; this is a manual run story. If latency exceeds 30 seconds consistently, document it and flag as post-MVP rather than blocking the epic. Boundaries: do NOT create gold entries or modify the skill or subagent during this story; open a fix story if bugs are found. Design doc ref: `docs/product/epics/MVP-1/design.md` S6.
