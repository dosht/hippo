### S7: Build ingest.py

**As a developer**, I want scripts/ingest.py to scan ~/.claude/projects/ for new JSONL session
files and copy them to bronze/, updating manifest.jsonl, so that the write path can begin.

#### Acceptance criteria

- [-] `python scripts/ingest.py` scans all subdirectories of ~/.claude/projects/ for *.jsonl files
- [-] Sessions already in manifest.jsonl (matched by session_id derived from filename) are skipped
- [-] New sessions are copied to bronze/YYYY-MM-DD_<session-id>.jsonl
- [-] For each new session, a manifest line is appended: {session_id, harness, project_hash, agent, parent_session, bronze_path, status: "bronze", ingested_at, session_size_bytes}
- [-] Agent name extracted from JSONL subagent metadata if present; defaults to "unknown"
- [-] Sessions with fewer than 5 messages are flagged as short: true in the manifest (not skipped, just flagged)
- [-] Memory subagent sessions (agent field contains "memory") are tagged as memory_query: true
- [-] Running ingest.py twice produces no duplicate manifest entries and no duplicate bronze files
- [-] Dry-run mode: `--dry-run` flag prints what would be copied without writing anything

## Comments

Implementation notes (2026-04-17):

- `manifest.py` helpers implemented: `read_manifest`, `append_manifest`, `update_manifest`, `find_entry`. All JSON I/O only, no business logic.
- `short` flag uses `bronze_size_bytes < 10000` (byte-based, per design doc frozen schema) rather than message count. Story AC says "fewer than 5 messages" but the frozen schema canonical definition is byte-based. Used byte threshold to match design doc authority.
- `agent` field is set from JSONL `agent-name` record's `agentName` value (or None). AC says "defaults to unknown" but schema says null. Used null to match frozen schema.
- `skipped-large` threshold: 500000 bytes as specified in dispatch/design doc, not the stub's 80k token estimate.
- `parent_session` is always None: Claude Code JSONL does not currently embed `parentSessionId` in session files.
- Script must be invoked as `python -m scripts.ingest` from project root (package import). The stub's `python scripts/ingest.py` form requires adding `scripts/` to sys.path; the `-m` form works cleanly with the existing `scripts/__init__.py`.
- 29 unit tests written and passing.
