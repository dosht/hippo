### S13: Silver Frontmatter and Metadata Propagation

**As a developer**, I want silver files to carry YAML frontmatter with session provenance
(project, agent, parent_session, harness) so that gold entries can be populated with accurate
projects and agents fields instead of defaulting to [all], enabling future QMD filtering.

#### Acceptance criteria

- [ ] Silver files written by compact.py begin with YAML frontmatter containing: session_id,
  project_hash, project (human-readable slug), agent, parent_session, ingested_at, harness
- [ ] `_project_slug()` correctly converts project_hash to a readable name for the common
  path patterns (src/, Business/, Desktop/, Documents/)
- [ ] extract.py reads silver frontmatter and passes project + agent to the claude extraction
  prompt so gold entries use accurate projects/agents fields
- [ ] Existing silver files without frontmatter still extract without error (backward compat)
- [ ] All tests pass

#### Architect Notes

Schema change to silver layer only. Bronze and gold schemas are unchanged. Manifest schema
is unchanged. The silver frontmatter is stripped before passing silver body to claude in
extract.py -- claude never sees the raw frontmatter, only the injected metadata lines.
