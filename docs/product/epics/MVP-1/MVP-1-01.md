### S1: Seed Gold Entries (Unblocks Read-Path Testing Immediately)

**As a developer**, I want 5-10 hand-crafted gold entries in gold/entries/ so that I can test the
read path (QMD, memory subagent, hippo-remember) without waiting for the pipeline to be built.

#### Acceptance criteria

- [ ] At least 5 .md files in gold/entries/ (not counting the sample template)
- [ ] Each file has all required YAML frontmatter fields (id, type, topics, projects, agents, confidence, source_sessions, created, last_validated, last_queried, query_count, staleness_policy, supersedes)
- [ ] Entries cover at least 3 different types (operational, pattern, gotcha, or decision-context)
- [ ] Entries are drawn from real past sessions (CDP port, Railway deploy, Supabase RLS, or similar concrete operational knowledge Mustafa has encountered)
- [ ] No em dashes in any entry body or frontmatter

#### Architect Notes

Wave 1 (parallel with S7). No shared infrastructure stubs to flesh out; write directly into `gold/entries/*.md` using `gold/sample-gold-entry-format.md` as the canonical template. Boundaries: do NOT touch `.claude/agents/memory.md` (S4) or `scripts/` (S7). Include at least one `gotcha` entry with a near-miss failure; this type is the core value proposition and is required for the S8.5 quality gate.
