### S9: Build extract.py

**As a developer**, I want scripts/extract.py to read silver summaries and produce gold entries
with valid frontmatter, running duplicate detection before writing, so that the knowledge base
grows without accumulating redundant entries.

#### Acceptance criteria

- [ ] `python scripts/extract.py` reads manifest entries with status "silver"
- [ ] For each session, calls `claude -p --model claude-sonnet-4-5 --max-turns 1 --no-mcp` with the extraction prompt from docs/playbooks/extract.md
- [ ] Before writing each proposed entry, runs `qmd query "{proposed id and topics}" --collection hippo --json -n 3` and skips entries that are near-duplicates (similarity score above 0.85)
- [ ] Each written gold entry has all required frontmatter fields; confidence defaults to "medium" for auto-extracted entries
- [ ] source_sessions field populated with the session_id from manifest
- [ ] Manifest entry updated: status "gold", extracted_at timestamp, gold_paths list of absolute paths to written gold entry files
- [ ] If claude -p returns "No new knowledge to extract", the session is marked status "gold" with gold_paths: []
- [ ] After writing all entries for a run, `qmd update && qmd embed` is called automatically
- [ ] Running extract.py twice produces no additional gold entries (idempotent)

#### Architect Notes

Wave 3b (after S8.5 go/no-go gate). Stubs to flesh out: `scripts/extract.py` and `docs/playbooks/extract.md`. The duplicate detection threshold (0.85) is frozen in `DUPLICATE_THRESHOLD` in the stub; do NOT change it without updating the design doc. Call `qmd update && qmd embed` once at the end of each run, NOT per entry; this must use the same collection name (`hippo`) and mask as S2 and the S3 post-merge hook. Boundaries: do NOT modify `scripts/compact.py` or `scripts/manifest.py` signatures. Gotcha: use `manifest.py` helpers for all JSONL I/O. Field name: the manifest field is `gold_paths` (list of absolute paths to written gold entry files), not `gold_entries` (IDs). Design doc ref: `docs/product/epics/MVP-1/design.md` S9.
